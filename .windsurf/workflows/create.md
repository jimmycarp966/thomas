# /create - Create Application

Este workflow inicia el proceso de creación de una nueva aplicación o funcionalidad.

## Proceso

### 1. Análisis de la Solicitud
- Entender qué quiere el usuario
- Si falta información, hacer preguntas claras

### 2. Planificación del Proyecto
- Determinar el stack tecnológico
- Planificar la estructura de archivos
- Crear plan detallado

### 3. Construcción de la Aplicación (Después de Aprobación)
- Coordinar con expertos:
  - `database-architect` → Schema
  - `backend-specialist` → API
  - `frontend-specialist` → UI

### 4. Preview
- Iniciar preview cuando esté completo
- Presentar URL al usuario

## Ejemplos de Uso

```
/create blog site
/create e-commerce app with product listing and cart
/create todo app
/create Instagram clone
/create crm system with customer management
```

## Antes de Empezar

Si la solicitud no está clara, haz estas preguntas:
- ¿Qué tipo de aplicación?
- ¿Cuáles son las funcionalidades básicas?
- ¿Quién la usará?

Usa valores predeterminados, agrega detalles después.

## Checklist

Antes de empezar a construir:
- [ ] Entender completamente la solicitud
- [ ] Identificar el stack tecnológico apropiado
- [ ] Crear plan de implementación
- [ ] Obtener aprobación del usuario
- [ ] Coordinar con los especialistas apropiados
